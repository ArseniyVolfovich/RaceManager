# Практическая работа № 21

## Работа с Git и GitHub

**Программный продукт:** веб-приложение RaceManager  
**Удалённый репозиторий:** <https://github.com/ArseniyVolfovich/RaceManager>  
**Основная ветка:** `main`  
**Дата выполнения:** 02.07.2026

## Выполнение практической работы

### 1. Проверка Git

Для проверки установленной версии Git выполнена команда:

```bash
git --version
```

### 2. Проверка удалённого репозитория

Проект RaceManager связан с репозиторием GitHub:

```bash
git remote -v
```

Результат:

```text
origin  https://github.com/ArseniyVolfovich/RaceManager.git (fetch)
origin  https://github.com/ArseniyVolfovich/RaceManager.git (push)
```

### 3. Проверка состояния проекта

```bash
git status
```

Команда позволяет увидеть текущую ветку, изменённые файлы и состояние синхронизации с GitHub.

### 4. Добавление файлов

```bash
git add .gitignore ИТК30_Git.md
```

Подготовленные изменения проверены командой:

```bash
git diff --cached
```

### 5. Создание коммита

```bash
git commit -m "docs: Практика 21"
```

### 6. Отправка изменений на GitHub

```bash
git push origin main
```

После выполнения команды добавленные файлы и новый коммит появились в удалённом репозитории.

### 7. Просмотр истории

```bash
git log --oneline --decorate --graph -5
```

В истории отображается коммит:

```text
docs: Практика 21
```

Синхронизация локальной и удалённой веток проверяется командами:

```bash
git fetch origin
git status
```

## Вывод

Проект RaceManager размещён на GitHub. Локальный репозиторий связан с удалённым, изменения сохранены отдельным коммитом и отправлены в ветку `main`. История изменений доступна через команду `git log` и на странице репозитория GitHub.
